# 医学合规边界规则

> 供 `drug-medical-postlaunch` Skill 在 L8(合规与边界) 环节引用，同时供 `drug-medical-prelaunch` 参照。
> 用途：明确 promotional/non-promotional 边界、off-label 交流规则、IIS 合规、数据透明度要求。
> 注：本文件只给医学合规边界框架，全面合规主责合规/法务部门(见 SKILL §8 边界声明)。

## 1. Promotional vs Non-promotional 边界

### 1.1 核心区别
| | Promotional（推广性） | Non-promotional（非推广性） |
|---|---|---|
| 目的 | 推动处方/销售 | 科学交流/洞察收集 |
| 执行者 | 销售代表/市场部 | MSL/医学部 |
| 内容 | on-label 产品信息 | 科学数据/指南/机制/病例 |
| 合规要求 | 严格 on-label，禁止 off-label | 可讨论 off-label 科学信息（有限条件） |
| 考核 | 处方量/纯销 | KOL 互动质量/科学产出 |
| 记录 | 推广活动记录 | 学术互动记录 |

### 1.2 MSL 拜访 vs 销售代表拜访
| 维度 | MSL 拜访 | 销售代表拜访 |
|---|---|---|
| 目的 | 科学交流、洞察收集、观念影响 | 处方推动、销量达成 |
| 内容 | 数据/指南/机制/病例 | 产品优势/适应症/促销 |
| 合规 | non-promotional，可讨论 off-label 科学信息（有限制） | 严格 on-label，禁止 off-label |
| 考核 | KOL 互动质量/科学产出 | 处方量/纯销 |
| 信息传递 | 不能把 KOL 洞察违规传递给销售用于推广 | 不能把推广话术通过 MSL 传递 |

### 1.3 边界红线
- MSL 不能带推广任务（不带销量指标、不发推广物料）
- MSL 不能代替销售做推广拜访
- 销售不能借 MSL 名义做 off-label 推广
- 市场推广物料与 MSL 科学材料须分开管理
- 跨部门信息传递须合规（KOL 洞察可以结构化反馈策略层，但不能直接用于推广话术设计）

## 2. Off-label 交流规则

### 2.1 谁能讨论 off-label
| 角色 | 能否讨论 off-label | 条件 |
|---|---|---|
| MSL | ✅ 可以（有限条件） | 科学交流目的 / 非推广场景 / 有证据支撑 / 回应 KOL 主动提问 |
| 销售代表 | ❌ 严禁 | 任何场景都禁止 off-label 推广 |
| 市场部 | ❌ 严禁 | 推广物料必须 on-label |
| KOL | ✅ 可以 | KOL 作为医疗专业人员可自主讨论（公司不能引导） |

### 2.2 MSL 讨论 off-label 的条件
- **回应性**：回应 KOL 主动提出的科学问题，不是主动发起 off-label 讨论
- **科学目的**：以科学交流为目的，不是推动 off-label 处方
- **有证据**：须有发表的科学数据支撑（不能凭空讨论）
- **非推广场景**：不在推广活动中讨论（学术拜访/科学会议可以）
- **记录**：须记录讨论内容和科学依据

### 2.3 Off-label 红线（绝对禁止）
- 主动向 KOL 推荐 off-label 使用
- 在推广活动中讨论 off-label
- 用 off-label 信息设计推广话术
- 销售代表任何形式的 off-label 推广
- 资助 KOL 做 off-label 推广活动

## 3. IIS 合规

### 3.1 IIS 审核标准
| 维度 | 审核要点 |
|---|---|
| 科学性 | 研究问题有无科学价值 / 设计是否合理 / 能否产出有意义数据 |
| 合规性 | 是否变相推广 / 利益冲突 / 伦理审查 |
| 价值 | 对产品证据补充的价值 / 对医学策略的贡献 |
| 可行性 | 研究者能力 / 时间线 / 预算合理性 |

### 3.2 IIS 合规红线
- 不能以 IIS 名义做变相推广（资助 KOL 做有利于产品的"研究"）
- 不能通过 IIS 影响 KOL 处方行为（利益交换）
- 研究资助须有真实科学价值，不是"请人站台"
- 利益冲突须申报管理
- 研究结果须透明（不能只发表有利结果）

### 3.3 IIS 管理流程
```
申请接收 → 科学性审核 → 合规审核 → 合同签署 → 进度追踪 → 结果收集 → 发表支持
```

## 4. 数据透明度

### 4.1 发表完整性
- 不能只发表阳性结果而隐藏阴性结果
- 临床试验须在公开平台登记（ClinicalTrials.gov / 中国临床试验登记平台）
- 研究结果须在合理时间内公开发表
- 发表须完整呈现数据（不能选择性引用）

### 4.2 临床试验透明
- 上市前临床研究须登记注册
- 研究方案须公开（方案注册）
- 研究结果须公开（结果注册）
- 上市后研究同样适用

### 4.3 科学诚信
- 数据不得篡改、伪造
- 作者署名须真实（不能" ghost authorship"）
- 利益冲突须披露
- 研究资助来源须公开

## 5. 与销售/市场互动边界

### 5.1 信息传递规则
| 信息类型 | 可否传递给销售/市场 | 条件 |
|---|---|---|
| KOL 洞察（诊疗习惯/观念） | ✅ 可结构化反馈策略层 | 不能直接用于设计推广话术 |
| 竞品情报 | ✅ 可共享 | 须事实性，不带推广引导 |
| 核心医学信息 | ✅ 给市场部制作物料 | 须经医学审校 |
| 推广数据点 | ✅ 给销售一线使用 | 严格 on-label |
| KOL 个人信息 | ❌ 不能传递 | 隐私保护 |
| MSL 与 KOL 的私人交流 | ❌ 不能传递 | 信任保护 |

### 5.2 保持科学独立性
- 医学可以对产品数据做客观评价，包括指出不足
- 医学不是市场/销售的工具人，是科学把关者
- 医学审校有权否决不合规的推广物料
- 跨部门支持有 SOP，避免越界

## 6. 常见合规陷阱
- MSL 带推广任务 → 违反 non-promotional 原则
- 销售借 MSL 名义做 off-label → 双重违规
- IIS 变相推广 → 合规风险 + 信任崩塌
- 只发表阳性结果 → 数据透明度违规
- 核心信息无证据出处 → 推广物料合规不过
- 推广数据点 off-label → 销售违规
- KOL 洞察直接用于推广话术 → 信息传递违规
- 医学审校变成代写推广文案 → 角色越界
