# 领域医学洞察与竞品分析方法论

> 供 `drug-medical-postlaunch` Skill 在 L5(领域医学洞察与竞品分析) 环节引用。
> 用途：统一竞品医学情报收集、对比分析、核心信息包/推广数据点输出、弱点应对的方法论。
> 这是医学部最直接给商业"输送弹药"的模块——把已有临床数据拆解、对比、提炼成各部门能用的内容。

## 1. 竞品医学情报收集

### 1.1 情报维度
| 维度 | 收集内容 | 来源 |
|---|---|---|
| 研究进展 | 竞品新研究启动 / 研究完成 / 数据发表 | ClinicalTrials.gov / 期刊 / 会议 |
| 获批动态 | 新适应症获批 / 说明书变更 / 新剂型 | NMPA·FDA / 官方公告 |
| 指南变化 | 竞品在指南中的推荐变化 | 指南原文 / 学会公告 |
| 学术发声 | KOL 对竞品的评价 / 会议讨论 | 学术会议 / KOL 洞察 |
| 市场动态 | 竞品处方量 / 市场份额变化 | 行业报告 / IMS / 销售反馈 |

### 1.2 情报更新频率
- 竞品研究进展：季度更新
- 获批动态：实时追踪（重大变化随时更新）
- 指南变化：年度 + 更新版发布时
- 定期输出：竞品医学情报简报（季度）

## 2. 对比矩阵

### 2.1 对比维度框架
```
对比矩阵维度：
├── 疗效
│   ├── 主要终点结果（有效率/达标率/缓解率）
│   ├── 起效速度（起效时间）
│   └── 持续时间（疗效维持）
├── 安全性
│   ├── 总体不良事件发生率
│   ├── 严重不良事件
│   ├── 特定不良事件（机制相关/靶点相关）
│   └── 禁忌症
├── 特殊人群
│   ├── 儿童/老年
│   ├── 肝肾功能不全
│   └── 妊娠/哺乳
├── 依从性/给药便利性
│   ├── 给药途径（口服/注射/吸入）
│   ├── 给药频率（日/周/月）
│   └── 储存条件
├── 证据等级
│   ├── RCT 数量与质量
│   ├── RWE 证据
│   ├── 指南推荐级别
│   └── head-to-head 证据
└── 其他
    ├── 适应症范围
    ├── 药物相互作用
    └── 专利到期时间
```

### 2.2 对比类型与局限标注
| 对比类型 | 说明 | 局限标注要求 |
|---|---|---|
| head-to-head(直接比较) | 同一研究内直接比较两药 | 最强证据；须注意研究设计 |
| 间接比较(cross-study) | 不同研究间的数据对比 | **必须标注**：人群不同/终点定义不同/统计方法不同/研究质量差异 |
| 网状Meta | 多研究间接比较的统计方法 | 须标注假设前提和不确定性 |

**强制规则**：间接对比不得当确定结论呈现，必须标注"间接对比，存在局限"。

## 3. 医学核心信息包（给市场部）

### 3.1 什么是核心信息
- Key Medical Messages：推广时可讲的核心科学主张
- 每条主张有临床数据支撑、有证据出处、有适用边界
- 是市场部制作推广物料的科学源头

### 3.2 输出格式（每条）
```
核心信息 N：
├── 科学主张（一句话，可传播）
├── 支撑证据
│   ├── 研究名称（NCT号如有）
│   ├── 研究设计（RCT/单臂/RWE，样本量N=）
│   ├── 关键结果（具体数据 + 统计显著性）
│   └── 证据等级（见 evidence-levels.md）
├── 适用边界（能讲什么场景/不能讲什么）
├── 合规限制（on-label / off-label 标注）
└── 参考来源（期刊/指南/会议，标🔍）
```

### 3.3 核心信息筛选标准
- 有强证据支撑（至少 RCT 级别）
- 与产品定位一致（支撑核心价值主张）
- 有差异化（vs 竞品的优势点）
- on-label（在获批范围内）
- 不夸大（数据说什么讲什么）

### 3.4 数量建议
- 核心信息 3-5 条（不宜过多，聚焦关键主张）
- 每条都要经得起 KOL 质疑（有数据、有出处）

## 4. 推广数据点（给销售部）

### 4.1 与核心信息的区别
| | 医学核心信息（给市场） | 推广数据点（给销售） |
|---|---|---|
| 层级 | 科学主张层面（3-5条主张） | 具体到哪个数字、怎么讲 |
| 用户 | 市场部（做物料/策略） | 销售代表（一线拜访） |
| 内容 | 主张+证据+边界 | 具体数字+话术边界+合规提示 |
| 颗粒度 | 中（主张级别） | 细（一句话能讲的数据） |

### 4.2 输出格式（每条）
```
推广数据点 N：
├── 可讲数据（具体数字，如"研究中XX%患者达标"）
├── 来源（研究名称/期刊/指南，标🔍）
├── 话术建议（可以怎么讲，一句话）
├── 合规边界
│   ├── ✅ 能讲（on-label 范围内）
│   ├── ⚠️ 慎讲（需注意措辞，不能引申）
│   └── ❌ 不能讲（off-label / 无数据支撑）
└── 常见违规场景提醒（销售容易讲错的点）
```

### 4.3 推广数据点红线
- 严格 on-label：不能讲获批范围外的适应症/人群/用法
- 不能夸大：数据说 30% 不能讲"大部分"
- 不能暗示 superiority：没有 head-to-head 不能说"优于竞品"
- 不能 off-label：销售代表严禁 off-label 推广（MSL 可有限讨论科学信息，但销售不行）
- 每个数据点须有溯源出处

## 5. 弱点识别与应对

### 5.1 弱点类型
| 弱点类型 | 表现 | 应对方向 |
|---|---|---|
| 疗效短板 | 某些亚组疗效不如竞品 | 补亚组 RWE / 限定目标人群 / 生成 head-to-head |
| 安全性信号 | 特定不良事件发生率高 | 生成风险管理证据 / 说明书优化 / 患者筛选策略 |
| 证据不足 | RCT 数量少 / RWE 缺失 | 补 RWE / IIS / 发布策略加强 |
| 适应症窄 | 获批适应症少于竞品 | 适应症拓展临床计划 / 上市后研究 |
| 给药不便 | 频次高/途径不便 | 剂型优化建议(研发协同) / 依从性教育 |
| 竞品时间窗口 | 竞品即将上市/专利到期 | 独占期内建"循证+指南+KOL+品牌"四件套 |

### 5.2 弱点应对框架
```
弱点识别 → 影响评估（多大影响？哪些场景？）→ 应对方案 → 执行主体 → 时间线
```

### 5.3 弱点处理原则
- 弱点不是藏着，是识别后想应对方案
- 承认弱点比被发现弱点更可信（对 KOL 和准入谈判都是）
- 弱点应对须落地到具体行动（补什么研究 / 调什么策略 / 限什么人群）

## 6. 价值证据包（给准入部门）

### 6.1 内容框架
```
价值证据包
├── 产品核心价值主张
├── 临床优势证据（vs 标准治疗的疗效/安全性）
├── 竞品对比证据（head-to-head 或间接对比，标注局限）
├── 真实世界证据（RWE 补充 RCT）
├── 经济学证据（ICER/预算影响，与准入药经协同）
├── 指南推荐地位
└── 未满足需求论证
```

### 6.2 与准入协同
- 医学提供临床证据，准入负责谈判策略和药经模型（见准入 skill 第三柱 HEOR）
- 价值证据包是医学→准入的接口产出物

## 7. 常见陷阱
- 竞品情报只收集不分析 → 数据堆砌无策略价值
- 间接对比不标注局限 → 被竞品或审评质疑可信度
- 核心信息没有证据出处 → 市场部无法使用 / 合规不过
- 推广数据点 off-label → 销售违规风险
- 弱点只识别不应对 → 问题始终是问题
- 只给市场核心信息，忘了给销售具体数据点 → 销售一线不知道怎么讲
- 价值证据包与准入脱节 → 准入谈判时证据不够用
