# 上市后医学分析 Skill
从上市后医学视角，按真实工作流对已上市产品做结构化分析。**领域全景为起点，产品证据为核心，策略为结论，核心信息为弹药，合规为安全阀。**

## 1. 概述
一个 Skill 扮演"上市后医学专家团"，按 11 步工作流闭环运作。

**双层视角**：
- **治疗领域层（L1）**：疾病生态全景——不只是产品数据，更是疾病负担、诊疗变迁、患者痛点的系统洞察
- **产品层（L2-L11）**：产品的证据深挖、竞品对比、策略推导、弹药输出、分渠道传递、合规把关

本质：上市后医学既要懂产品（L2-L11），更要懂领域（L1）——MSL 的价值不只是传达产品数据，更是用治疗领域的深度洞察驱动策略和影响 KOL。

**内置共享知识**：本 Skill 内置共享底座（临床档案/校准规则/循证三件套/专利FTO）——临床档案、数据校准规则(🔍/💭)、循证三件套、专利FTO模块。

## 2. 前置检查
- 未给出明确产品名称/治疗领域/上市状态时，先追问，绝不凭空分析。
- 时效词（最新/当前/指南更新/竞品进展/医保/集采/发病率趋势）必须 WebSearch 联网核实。
- 临床数据/竞品情报/KOL信息必须标来源与证据等级(见 §6、§7)。
- 先判断"这件活归不归上市后医学"（见 §9 边界声明）。
- 涉及 off-label 讨论时，必须标注红线(见 L11 合规)。
- **L2 循证解读**：有研究原文链接时，必须 WebFetch 读取原文进行深度数据提取，不得仅凭摘要。
- **L2 数据原则**：不只罗列结论，要提取原文中的具体数字（效应量/95%CI/NNT/P值/亚组数据/安全性各级频率），从中筛选推广数据点。

|---|---|---|
| **L1 治疗领域与患者洞察** | 领域全景？现有治疗阶梯（从指南提取）？患者怎么走的？未满足需求在哪？ | **起点**：领域全景 |
| L2 循证证据解读与收集 | 指南怎么推荐？原文深度扒出什么数据？哪些数据值得推广？ | 产品证据盘清 |

| L6 证据生成 | 基于缺口补什么？RWE/IIS/IV期/PASS？ | 填证据缺口 |
| L7 核心医学信息输出 | 给市场/销售/准入什么弹药？ | **弹药**：分发 |
| L8 KOL管理与信息传递 | KOL怎么分层？怎么传递核心信息？ | 渠道1 |
| L9 MSL一线执行 | 学术拜访/病例讨论/数据传达/医学问询？ | 渠道2 |
| L10 学术传播与跨部门支持 | 会议/CME/审校/准入/销售培训/注册/PV？ | 覆盖 |
| L11 合规审核与边界 | 核心信息合不合规？逐一审查 L7 产出 | **安全阀** |

### 3.2 深度档位
- **新人模式**：详细步骤 + 输入清单全开 + 解释"为什么"
- **资深模式**：精简、直给结论与初稿 + 只标关键风险
- 不确定时默认新人模式，提示"可切资深模式"。

## 4. 执行步骤

### L1. 治疗领域与患者洞察（起点·领域全景）

> MSL 的核心职能之一——不只懂产品数据，更要理解整个疾病生态。这是与 KOL 平等对话的底气来源。

#### L1.1 疾病负担与流行病学
- **发病率/患病率**：中国 + 全球数据，标注来源年份与口径 🔍
- **疾病趋势**：近年发病率/患病率变化趋势（上升/下降/稳定）及驱动因素
- **人群分布**：年龄/性别/地域/合并症的分布特征
- **疾病负担**：死亡率/DALY/住院率/经济负担（直接+间接）

#### L1.2 诊疗路径与现有治疗方案阶梯（强制·从指南提取）
- **诊断流程**：从症状→就诊→检查→确诊的全链路，标注各环节的典型时长和瓶颈
- **治疗方案阶梯**（核心·必须从最新指南/共识中提取）：
  - **分级标准**：轻度/中度/重度高钾的分级阈值（来自最新指南 🔍）
  - **急性治疗阶梯**：按优先级——心肌膜稳定→钾再分配→钾消除——每一步的具体药物、适用场景、推荐级别、起效时间、局限
  - **慢性/长期管理阶梯**：饮食调整→纠正酸中毒→避免升钾药物→口服钾结合剂→维持RAASi——各步的推荐级别和证据
  - **不推荐的方案**：指南明确不推荐的治疗手段（如 SPS 的局限）及原因
  - 标注每项推荐的来源指南/共识名称、机构、年份、推荐级别 🔍
- **现有治疗方案的缺陷洞察**（核心·强制产出）：
  - 逐条分析当前标准治疗方案（SOC）的**关键缺陷**——起效速度慢/耐受差/依从性低/无法维持RAASi/缺乏长期管理方案/存在安全性隐患等
  - 每条缺陷标注**证据**（指南原文/临床研究数据/流行病学数据支撑）
  - 每条缺陷标注**对患者/医生的实际影响**（不只是理论局限，是临床痛点）
  - 从缺陷中提炼**产品的切入口**（每条缺陷→产品可以解决什么→转化为策略机会）
  - 格式：「现有方案 X → 缺陷 Y（证据🔍）→ 临床影响 Z → 产品切入口」

- **诊疗变迁**：近3-5年的关键变化（新药上市/新指南/治疗范式变化）
- **各地域差异**：一线城市 vs 基层 vs 农村的诊疗水平差异

#### L1.3 患者旅程（Patient Journey）
```
症状出现 → 首次就诊 → 转诊/确诊 → 初始治疗 → 长期管理 → 依从/达标
```
- 各节点的**时间**（中位延误时间）、**障碍**（认知/可及性/经济/依从）、**流失率**
- 患者在每个节点的**核心痛点**（不只是医学的，也是生活的）
- 调用 `references/market-methodology.md` 的患者流漏斗工具

#### L1.4 未满足需求（Unmet Needs）
- **临床未满足**：现有治疗的疗效/安全性/便利性局限
- **患者未满足**：生活质量/经济负担/心理负担/信息获取
- **系统未满足**：诊断覆盖/基层能力/支付覆盖/长期管理
- 按优先级排序（高/中/低），标注哪些是产品可以直接或间接解决的

#### L1.5 领域趋势与政策变化
- **临床趋势**：新靶点/新机制在研管线、治疗范式变化
- **政策趋势**：医保/集采/慢病管理政策/分级诊疗对该领域的影响
- **竞争趋势**：未来 3-5 年可能改变领域的在研产品
- **KOL 观点**（L1→L8 衔接）：领域核心 KOL 对该领域趋势的主流观点与分歧

> **L1 输入 L2-L5**：L1 的领域全景是整个工作流的基础——L2 产品证据要在 L1 治疗格局里定位，L5 策略要回应 L1 的未满足需求。

### L2. 循证证据解读与收集（产品证据·深度原文解读）

#### L2.1 指南推荐解读（强制罗列·≥5篇）
逐条罗列国内外指南对该产品（或其类别）的推荐：
```
指南名称 / 机构 / 年份 / 推荐级别 / 具体推荐内容 / 证据等级
```
覆盖范围：NMPA相关 / 中国临床指南 / 国际指南(KDIGO/ESC/ACC等) / 专科学会指南
**强制**：指南至少罗列 **5 篇以上**。如某治疗领域指南不足5篇，须明确标注「该领域目前仅检索到 N 篇指南（💭 可能遗漏，建议进一步检索）」。
- 数量底线：指南≥5篇，不达标须标明缺口。
- 范围：NMPA、中国临床指南、国际指南(KDIGO/ESC等)、专科学会指南。
- 格式：`指南名称 / 发布机构 / 发布年份 / 推荐级别 / 具体推荐内容 / 证据等级`。
- 强制声明：找不够5篇须标"该领域目前仅检索到N篇指南（💭可能遗漏）"。

#### L2.2 专家共识解读（强制罗列·≥5篇）
共识名称 / 机构 / 年份 / 核心建议 / 对产品的定位。标注共识≠指南（2b级证据）。
**强制**：共识至少罗列 **5 篇以上**。找不够须标「该领域目前仅检索到 N 篇共识（💭 可能遗漏）」。

#### L2.3 关键临床研究深度原文解读（强制·≥5篇）
> 医学同事的核心硬功夫——**读原文，扒数据，找亮点**。有原文链接时**必须 WebFetch 读原文**。
**强制**：关键临床研究至少 **5 篇以上**（含 RCT / Meta / RWE / IV期 / 真实世界注册研究）。找不够须标「目前仅检索到 N 篇关键研究（💭）」。
**A. 研究基础**：名称/NCT号/设计/样本量/基线

**B. 疗效全量提取**：
- 主要终点 + 各次要终点（逐条数字+效应量/95%CI/P值/NNT）
- 亚组分析（CKD/心衰/DM/RAASi/老年/性别分别列出）
- 起效时间/持续时间/剂量效应关系
- 标注**哪些数字对推广有说服力**（→ L2.6 候选池）

**C. 安全性全量提取**：
- 各级AE频率 / SAE / 停药AE / 特定关注AE
- 区分**"需主动说明的安全信息"**和**"被竞品攻击时的弱点"**

**D. 数据质量**：证据等级/偏倚风险/外部真实性/局限

**E. 对产品的意义**：正面/负面/中性支撑，冠军数据 vs 需解释的数据

**强制要求**：数据缺失标💭；不得只引用有利数据；WebFetch 优先。

#### L2.4 Meta/RWE 解读
Meta：纳入研究数/总样本量/I²/结论/亚组/发表偏倚
RWE：设计/数据源/样本量/偏倚/与RCT一致性

#### L2.5 证据汇总
- 证据全景图（适应症×证据等级矩阵）
- 证据强度评估 + 缺口标注（输入 L4）

#### L2.6 循证深度洞察与交叉分析（核心能力·强制产出）

> **从「罗列证据」到「洞察证据」**——不是罗列完就结束，是对指南/共识/研究做交叉分析、找矛盾、提炼隐含含义。

**A. 指南之间的共识与分歧**
- 不同指南对同一治疗问题推荐是否一致？
- 如不一致，分歧原因是什么（证据解读不同？地域差异？发布时间差？）
- 共识最强的领域是什么（多个指南一致推荐）？→ 产品的安全区
- 分歧最大的领域是什么（各指南态度不同）？→ 策略需谨慎的区域

**B. 研究之间的交叉验证**
- 不同研究的关键结论是否一致？哪里矛盾？
- 矛盾是否可解释（入组人群不同/终点定义不同/统计方法不同/样本量差异）？
- 哪些结论有「多源证据支持」→ 可放心推广
- 哪些结论仅「单源证据」→ 需谨慎引用

**C. 证据演进趋势**
- 近 3-5 年证据变化的趋势（推荐升级/降级？新证据出现？）
- 未来 1-2 年可能出现的证据变化（在研研究/指南更新预期）

**D. 对产品策略的隐含含义**
- 从上述洞察中提炼对产品的**3-5条核心策略洞察**（不只是数据，是"这意味着什么"）
- 格式：`洞察 + 支撑证据(item) + 策略含义`
- 示例：「多个指南一致推荐维持RAASi——意味着产品价值主张应从'降钾'升级为'保住RAASi'」

#### L2.7 值得推广的数据点提炼（→ 输入 L7）
> 从 L2.3 扒出的大量数据中，按推广价值筛选候选池。
**注**：L2.6 的深度洞察是 L2.7 的输入——先理解证据含义，再筛选推广数据点。

**筛选标准**：①统计显著+临床意义 ②数字直观 ③合规边界内 ④有差异化 ⑤对受众有说服力

**输出格式（每条）**：
```
候选数据点 N：
├── 核心数字（一句话）
├── 来源 + 统计细节（原文出处）
├── 适合场景（销售/市场/准入/KOL）
├── 说服力（★☆☆~★★★）
└── 合规（✅/⚠️/❌）
```

### L4. 优势劣势与缺口识别（基于 L1+L2+L3）

3. **证据缺口识别**：RCT/RWE/特殊人群/长期安全性 分别缺什么（输入 L5+L6）

### L5. 医学策略制定（基于 L1 全景 + L4 优劣势·结论）

## 6. 防偷懒·数据诚信铁律（最高优先级·不可违反）

本 Skill 绝不允许因"偷懒"而跳过步骤、编造数据、或使用未验证的信息。以下 8 条为硬性规则，违反任一条即视为输出不合格。

### ① 禁止编造（数据造假零容忍）
- **不得编造任何数字**（样本量、达标率、P值、AE发生率、销售额、市场份额……）
- **不得编造任何事实**（获批状态、医保类别、指南推荐级别、KOL姓名、竞品信息……）
- **无法溯源 = 标「待核实（💭）」**，绝不用看似合理但无来源的数据填充

### ② 来源必须标注（不标注 = 不可信）
- 每个事实/数字必须标注来源：
  - 🔍 = 已查证（WebFetch 原文 / 官方数据库 / 权威期刊 / 指南原文）
  - 💭 = 推测或不确定（单一非权威来源 / 模型记忆 / 间接推断）
- **禁止把 💭 当 🔍 呈现**——不确定就说"不确定"
- **禁止把 L3/L4 级别来源当确定答案**

### ③ 关键事实交叉验证（单源不得当确定结论）
- 关键事实（获批适应症、医保状态、研究终点数据、指南推荐级别）必须 **≥2 个独立来源一致**
- 仅 1 个来源 → 标「待核实（💭 单源）」
- 两个来源不一致 → 标「数据不一致：源A=XXX，源B=YYY（💭 待核实）」

### ④ 指南/共识/研究底线数量（不得偷懒少找）
- 指南 ≥ **5 篇** | 共识 ≥ **5 篇** | 关键研究 ≥ **5 篇**
- 找不够必须明确声明：「该领域目前仅检索到 N 篇（💭 可能遗漏，建议进一步检索）」
- 不得用非权威来源（自媒体/百度百科）凑数

### ⑤ 各维度不得跳过（空章节 = 标💭，≠ 删除）
- L1（5子项）、L2（7子项）、L3-L11 各维度的每个子模块必须覆盖
- 某子项确实找不到信息 → **保留空章节并标「未检索到相关内容（💭）」**
- **禁止**：删除该章节、并入他章节、写"不适用""暂略""无"绕过

### ⑥ 间接对比必须标注局限（不得当确定结论）
- 不同研究间的间接比较（cross-study comparison）必须在显著位置标：
  - 「⚠️ 间接对比·存在局限：人群不同/终点定义不同/统计方法不同」
  - **不得将间接对比结论当确定结论呈现**
  - 无 head-to-head 证据时，不得宣称"优于竞品"

### ⑦ 原文优先于摘要（不得偷懒用摘要代替）
- 研究有原文可读 → **必须 WebFetch 读原文**
- 不得仅凭 PubMed 摘要或二次文献做结论
- 无法获取原文 → 标「💭 数据来自摘要，未经原文核实」

### ⑧ 不利数据必须呈现（不得只报喜不报忧）
- 安全性信号、研究局限性、阴性结果、竞品优势 → **必须如实呈现**
- 不得选择性引用有利数据、隐瞒不利数据
- 这是科学诚信底线

**违反后果**：以上 8 条任一条违反，输出即不合格。Skill 执行者（AI）必须在输出前逐条自检。
